# Checklist Completo do Protótipo Temporal de Campo para Identificação de Macroalgas

Este checklist consolida, item a item, o que precisa existir, ser conferido ou ser executado no projeto. Ele foi organizado a partir da metodologia, do app de campo, dos protocolos operacionais, do mapa infra e da publicação em GitHub Pages + Render.

## 1. Definição do projeto

- [ ] Nome do projeto padronizado em todos os documentos.
- [ ] Objetivo geral descrito de forma consistente.
- [ ] Escopo do protótipo delimitado para campo, validação e documentação.
- [ ] Versão pública do site verificada.
- [ ] Estrutura do repositório principal revisada.
- [ ] Estrutura do repositório de apoio de render revisada.

## 2. Base metodológica

- [ ] Metodologia do protótipo revisada e publicada em HTML.
- [ ] Desenho amostral descrito com 6 estações.
- [ ] Definição dos quadrados de 25 x 25 cm confirmada.
- [ ] Número de quadrados por estação confirmado em 9.
- [ ] Coleta em maré baixa de sizígia documentada.
- [ ] Coleta destrutiva da macroflora confirmada.
- [ ] Variáveis ambientais registradas definidas.
- [ ] Processamento posterior descrito.
- [ ] Sequência ideal por estação descrita.

## 3. Estações de estudo

- [ ] Ponta das Canas.
- [ ] Sambaqui.
- [ ] Ponta do Coral.
- [ ] Saco dos Limões.
- [ ] Ribeirão da Ilha.
- [ ] Caieira da Barra do Sul.
- [ ] Legendas históricas de estações conferidas.
- [ ] Legendas ambientais conferidas.
- [ ] Siglas e vocabulário padronizados.

## 4. Pré-campo

- [ ] Planejamento temporal definido.
- [ ] Reconhecimento prévio dos pontos feito.
- [ ] Desenho amostral padronizado.
- [ ] Rotulagem antecipada preparada.
- [ ] Material audiovisual preparado.
- [ ] Lista rápida antes de sair conferida.
- [ ] Condições de acesso aos pontos avaliadas.
- [ ] Checagem de maré e janela de coleta feita.
- [ ] Checagem de segurança e convivência feita.

## 5. Materiais obrigatórios

- [ ] GPS ou celular com geolocalização.
- [ ] Roteiro impresso ou digital da estação.
- [ ] Fita métrica ou cabo-guia.
- [ ] Quadrados de 25 x 25 cm.
- [ ] Etiquetas e canetas resistentes à água.
- [ ] Sacos e frascos para coleta.
- [ ] Material para preservação das amostras.
- [ ] Termômetro.
- [ ] Refratômetro ou equipamento equivalente para salinidade.
- [ ] Recipientes para água.
- [ ] Equipamentos de proteção individual.
- [ ] Câmera ou celular para fotos.
- [ ] Bateria carregada e power bank.

## 6. Segurança e preservação

- [ ] Equipamentos de segurança levados.
- [ ] Conduta ética e convivência lidas pela equipe.
- [ ] Risco de mar, pedras, escorregões e acesso avaliado.
- [ ] Plano de retorno e comunicação definido.
- [ ] Cuidados com preservação das amostras estabelecidos.
- [ ] Contingência para atraso no processamento definida.

## 7. App de campo

- [ ] App aberto no navegador principal.
- [ ] Registro de estação funcionando.
- [ ] Registro de quadrados funcionando.
- [ ] Rascunho local funcionando.
- [ ] Exportação de JSON funcionando.
- [ ] Importação de JSON funcionando.
- [ ] Sincronização com Supabase funcional.
- [ ] Sincronização entre abas validada.
- [ ] Sincronização entre computador e celular validada.
- [ ] Guia de uso do app disponível.
- [ ] Backup local confirmado como contingência.

## 8. Ficha de campo

- [ ] Identificação geral preenchida.
- [ ] Projeto preenchido.
- [ ] Equipe preenchida.
- [ ] Data e horários preenchidos.
- [ ] Estação selecionada.
- [ ] Coordenadas GPS registradas.
- [ ] Maré registrada.
- [ ] Temperatura da água registrada em 3 leituras.
- [ ] Salinidade registrada em 3 leituras.
- [ ] Transparência visual registrada.
- [ ] Hidrodinamismo observado registrado.
- [ ] Substrato predominante registrado.
- [ ] Declividade do costão registrada.
- [ ] Impactos visíveis registrados.
- [ ] Observações gerais preenchidas.

## 9. Amostragem quantitativa

- [ ] 9 quadrados por estação conferidos.
- [ ] Posições definidas por sorteio ou critério equivalente.
- [ ] Quadrados posicionados ao longo do cabo-guia.
- [ ] Espécies dominantes por quadrado registradas.
- [ ] Foto e escala associadas a cada quadrado.
- [ ] Coleta indicada em cada quadrado.
- [ ] Coleta complementar fora dos quadrados conferida.

## 10. Registro multimídia

- [ ] Fotos pré-campo feitas.
- [ ] Fotos da estação feitas.
- [ ] Fotos dos quadrados feitas.
- [ ] Vídeo de contexto feito quando necessário.
- [ ] Áudio/observações gravadas quando necessário.
- [ ] Roteiro de divulgação científica preservado.
- [ ] Gestão pós-campo dos arquivos feita.

## 11. Processamento de dados

- [ ] Material triado.
- [ ] Identificação taxonômica executada.
- [ ] Peso seco calculado quando aplicável.
- [ ] Dados ambientais consolidados.
- [ ] Base tabular revisada.
- [ ] Arquivos CSV finais gerados.
- [ ] Arquivos JSON de validação gerados.
- [ ] Checklist de publicação dos dados conferido.

## 12. GIS e mapas

- [ ] Mapa operacional funcionando.
- [ ] Mapa infra funcionando.
- [ ] Pontos de estudo conferidos no mapa.
- [ ] Camadas metodológicas conferidas.
- [ ] Legenda visual conferida.
- [ ] Exportação PNG validada.
- [ ] Imagem PNG final revisada.
- [ ] Layout sem sobreposição ou corte incorreto.

## 13. Render PNG em produção

- [ ] Serviço Render em produção ativo.
- [ ] Health check respondendo 200.
- [ ] PNG direto respondendo 200.
- [ ] Mapa publicado apontando para o endpoint live.
- [ ] Variável de ambiente ou fallback público configurado.
- [ ] Teste de exportação feito no navegador.
- [ ] Dependências do serviço de render estabilizadas.

## 14. Publicação web

- [ ] GitHub Pages ativo.
- [ ] Índice do portal funcionando.
- [ ] Metodologia navegável.
- [ ] App de campo acessível.
- [ ] Mapas acessíveis.
- [ ] Dashboard acessível.
- [ ] Links internos revisados.
- [ ] Caminhos relativos conferidos.
- [ ] Cache do navegador testado após atualização.

## 15. Divulgação e apresentação

- [ ] Legendas históricas revisadas.
- [ ] Material para apresentação preparado.
- [ ] PDF ou artigo de apoio referenciado.
- [ ] Imagens de apoio com autoria conferida.
- [ ] Narrativa do protótipo padronizada.
- [ ] Mensagem de impacto ambiental consistente.

## 16. Manutenção contínua

- [ ] Backup dos dados salvo.
- [ ] Checklist semanal executado.
- [ ] Logs de incidentes atualizados.
- [ ] Regressões no PNG revisadas.
- [ ] Mudanças em URLs públicas monitoradas.
- [ ] Atualizações do Render testadas após deploy.

## 17. Lista rápida antes de sair para campo

- [ ] Celular carregado.
- [ ] Power bank carregado.
- [ ] Internet confirmada.
- [ ] GPS funcionando.
- [ ] App de campo aberto.
- [ ] Mapa e health check favoritos salvos.
- [ ] Materiais físicos na mochila.
- [ ] Saída e retorno comunicados.

## 18. Lista rápida antes de publicar uma versão nova

- [ ] Links internos revisados.
- [ ] Mapa infra exportando PNG.
- [ ] Render live respondendo health.
- [ ] Documentação sem referência antiga.
- [ ] README atualizado.
- [ ] Protocolos atualizados.
- [ ] GitHub Pages atualizado.
